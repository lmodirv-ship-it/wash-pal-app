سأصلح مشكلة بقاء التطبيق على “جاري التحميل...” بعد الدخول، وأجعل الدخول بالإيميل وكلمة السر يوجّه كل مستخدم حسب دوره بشكل واضح.

المشكلة التي ظهرت من الفحص:
- آخر تسجيل دخول ناجح كان للمستخدم `taltnamart@gmail.com`.
- هذا المستخدم موجود في نظام الدخول، لكن لا يوجد له سجل في `profiles` ولا صلاحيات في `user_roles` ولا عضوية محل.
- الكود الحالي ينتظر `profile` بلا نهاية عند وجود user بدون profile، لذلك تظهر شاشة “جاري التحميل...” دائماً.
- Trigger إنشاء profile تلقائياً غير موجود حالياً على جدول المستخدمين، لذلك الحسابات الجديدة لا تُربط بملف وصلاحية تلقائياً.

الخطة:

1. إصلاح قاعدة البيانات للحسابات الحالية والجديدة
   - إعادة إنشاء Trigger آمن عند تسجيل أي مستخدم جديد لإنشاء:
     - `profiles`
     - `user_roles`
   - ترميم الحسابات الموجودة التي ليس لها profile أو role، وإعطاؤها الدور الافتراضي `employee` أو حسب البيانات الموجودة.
   - الحفاظ على حساب المالك `lmodirv@gmail.com` كـ `admin` وعدم إنزاله أو تغييره.

2. منع التعليق اللانهائي في الواجهة
   - تعديل `AuthContext` ليحفظ حالة خطأ/عدم وجود profile بدلاً من البقاء في loading.
   - إذا دخل مستخدم بدون profile، يتم محاولة إنشاء/استرجاع profile أو توجيهه لصفحة مناسبة بدلاً من “جاري التحميل...” بلا نهاية.

3. ضبط التوجيه بعد الدخول حسب الدور
   - `admin` → صفحة المالك `/admin`
   - `supervisor` أو `manager` → لوحة التحكم `/dashboard`
   - `employee` → صفحة الموظف `/employee`
   - `customer` → تطبيق العميل `/app`
   - إن كان المستخدم يحتاج محل ولا يوجد له محل، يذهب إلى `/create-shop` فقط عندما يكون ذلك مناسباً.

4. تنظيف صفحة الدخول
   - الإبقاء على الدخول بكلمة السر مباشرة فقط.
   - إزالة/تعطيل بقايا منطق الكاميرا أو magic-link من صفحة الدخول إذا كان غير مستخدم في التدفق الحالي حتى لا يسبب التباساً.
   - تحسين رسالة الخطأ إذا فشل تحميل الملف الشخصي.

5. إصلاح تحذير مكوّن تغيير اللغة
   - معالجة تحذير React الخاص بـ refs في `LanguageSwitcher`/Dropdown حتى لا تظهر أخطاء مزعجة في الكونسول أثناء صفحة الدخول.

6. التحقق بعد التنفيذ
   - فحص تسجيل الدخول للمالك والتأكد أنه يذهب إلى `/admin`.
   - فحص مستخدم عادي بلا profile سابق والتأكد أنه لا يعلق على “جاري التحميل...”.
   - التأكد أن الصفحات المحمية لا تفتح إلا حسب الصلاحيات.

ملاحظات تقنية:
- سيتم تنفيذ تغييرات قاعدة البيانات عبر migration آمن.
- لن يتم تعديل ملفات Lovable Cloud auto-generated مثل `src/integrations/supabase/client.ts` أو `types.ts`.
- سيتم إبقاء الأدوار الأساسية في جدول `user_roles` كما هو مطلوب أمنياً، مع إبقاء `profiles.role` فقط للتوافق مع الكود الحالي إلى أن يتم توحيده لاحقاً.
